# 项目说明

## 📋 整合内容

本项目将原本分散的8个独立HTML页面整合成一个统一的全屏滚动单页应用。

### 页面整合顺序

| 序号 | 页面名称 | 原始路径 | 展示内容 |
|------|---------|---------|---------|
| 1 | 烹饪方法桑基图 | `pengren.html` | 八大菜系与烹饪技法的流向关系 |
| 2 | 八大菜系溯源图谱 | `八大菜系溯源图谱.html` | 树形图+地图联动展示菜系源流 |
| 3 | 八大菜系历史演变 | `八大菜系历史演变.html` | 时间轴地图展示历朝历代菜系发展 |
| 4 | 八大菜系气泡图 | `烹饪方法/八大菜系气泡图.html` | 动态气泡展示菜品热度排行 |
| 5 | 各省菜系占比 | `烹饪方法/各省菜系占比可视化.html` | 全国地域分布与市场份额 |
| 6 | 口味经纬度图 | `烹饪方法/口味经纬度图.html` | 散点图分析菜系口味维度 |
| 7 | 八大菜系转盘 | `烹饪方法/八大菜系转盘.html` | 互动抽奖转盘选择菜系菜品 |
| 8 | 菜品多维数据 | `新建文件夹/菜品.html` | 雷达图展示代表菜品特征 |

## 🎯 整合逻辑

### 内容流程设计
```
首页介绍 → 历史溯源 → 时代演变 → 菜品分析 → 地域分布 → 口味特征 → 互动体验 → 菜品详情
```

### 叙事结构
1. **宏观认知（1-2页）**：从烹饪方法和菜系源流入手，建立整体认知框架
2. **历史纵深（3页）**：时间维度展现菜系发展演变
3. **数据分析（4-6页）**：从热度、地域、口味多角度分析菜系特征
4. **互动体验（7-8页）**：通过转盘和菜品详情提供沉浸式体验

## 🛠️ 技术实现

### 核心技术栈
- **fullPage.js 4.0.20** - 提供全屏滚动效果和页面管理
- **iframe嵌套** - 保持原有页面独立性和交互功能
- **统一导航** - 右侧导航点+键盘快捷键+鼠标滚轮

### 关键特性
1. **性能优化**：iframe懒加载减少首屏压力
2. **响应式设计**：适配不同屏幕尺寸
3. **流畅动画**：700ms缓动切换，easeInOutCubic曲线
4. **多端支持**：支持PC端、平板、触摸屏设备

### 页面通信
- 各子页面保持完全独立，无需修改原有代码
- 通过iframe隔离，避免样式冲突和脚本干扰
- 右侧导航栏提供统一的页面索引

## 📁 文件说明

### 核心文件
- `index.html` - 主入口文件，包含fullPage.js配置和页面框架
- `pages/` - 存放8个子页面的目录

### 辅助文件
- `README.md` - 详细的技术文档和使用说明
- `使用指南.txt` - 简明的快速上手指南
- `启动本地服务器.bat` - 一键启动Python HTTP服务器的批处理脚本
- `项目说明.md` - 本文件，说明整合逻辑和设计思路

## 🎨 设计理念

### 视觉统一
- 保持所有页面的宣纸背景风格
- 统一的朱砂红、墨色、古金色配色方案
- 右侧导航采用传统中式设计语言

### 交互体验
- 固定全屏，避免传统滚动的信息过载
- 每次滑动切换一整屏，类似PPT演示
- 支持多种输入方式（键盘、鼠标、触摸）

### 内容编排
- 遵循"总-分-总"的叙事逻辑
- 先宏观概览，再深入细节，最后互动体验
- 从静态数据到动态交互，渐进式提升参与度

## 🚀 部署方式

### 本地预览
```bash
# 方式1：双击打开
index.html

# 方式2：启动本地服务器（推荐）
启动本地服务器.bat

# 方式3：命令行启动
python -m http.server 8000
```

### 云端部署
支持一键部署到以下平台：
- GitHub Pages
- Vercel
- Netlify
- Cloudflare Pages
- 自建服务器（Nginx/Apache）

## 📊 数据来源

所有数据仅用于教学展示：
- 八大菜系历史资料：来自公开文献整理
- 菜品热度数据：模拟生成，仅供演示
- 地域分布数据：基于公开餐饮统计数据估算
- 口味维度数据：基于典型菜品特征归纳

## ⚠️ 注意事项

1. **网络依赖**
   - 所有外部库使用CDN加载
   - 中国地图数据从阿里DataV获取
   - 建议在稳定网络环境下访问

2. **浏览器兼容**
   - 推荐Chrome 90+、Firefox 88+、Edge 90+
   - 不支持IE浏览器
   - Safari需14+版本

3. **文件完整性**
   - 不要删除`pages/`目录下的任何文件
   - 不要修改`index.html`中的iframe路径
   - 部署时需保持整个文件夹结构

4. **跨域问题**
   - 直接打开index.html可能遇到iframe跨域限制
   - 强烈建议使用本地服务器方式运行
